## 1️⃣ $not — Exclude by Condition

`$not` is used when your logic is about **what to exclude**, not what to include.

**Example — Jobs NOT requiring more than 2 years experience:**
```js
db.jobs.find({
  experience: { $not: { $gt: 2 } }
})
```
> Returns: **Software Engineer (Startup)** — only 1 year experience.

**⚠️ Common Mistake:**
```js
// ❌ WRONG — $not outside field level
{ $not: { experience: { $gt: 2 } } }

// ✅ CORRECT — $not inside field condition
{ experience: { $not: { $gt: 2 } } }
```

> 💡 `$not` must always be applied **at the field level**, never at the query root.

---

## 2️⃣ $nor — Exclude Multiple Conditions

`$nor` excludes documents that match **any** condition in the array. Think of it as the **"none of these"** operator.

**Example — Jobs neither from Amazon nor in Bangalore:**
```js
db.jobs.find({
  $nor: [
    { company: "Amazon" },
    { location: "Bangalore" }
  ]
})
```
> Returns: **Software Engineer (Startup)** — not Amazon, not Bangalore.

**Logic:**
```
NOT (company = Amazon  OR  location = Bangalore)
```

> 💡 Perfect for building **exclusion filters** in real search systems.

---

## 3️⃣ $regex — Pattern Matching on Strings

`$regex` lets you search string fields using patterns.

**Starts with "Senior":**
```js
db.jobs.find({ role: { $regex: "^Senior" } })
```

**Ends with "Developer":**
```js
db.jobs.find({ role: { $regex: "Developer$" } })
```

**Case-insensitive search:**
```js
db.jobs.find({ role: { $regex: "backend", $options: "i" } })
```

**Pattern Reference:**

| Pattern | Meaning |
|---------|---------|
| `^` | Starts with |
| `$` | Ends with |
| `.*` | Any characters |
| `$options: "i"` | Case insensitive |

**⚠️ Performance Warning:**
> Regex does **not use indexes** unless the pattern starts with `^`. It scans the entire collection. Avoid regex in high-traffic APIs — use it only when absolutely necessary.

---

## 4️⃣ $expr — Field-to-Field Comparison

`$expr` is used when your condition depends on **comparing two fields** inside the same document — something a normal query cannot do.

**Normal query** — field vs fixed value:
```js
db.jobs.find({ experience: { $gt: 3 } })
```

**$expr** — field vs calculated value:

**Example 1 — Salary greater than experience × 5 lakh:**
```js
db.jobs.find({
  $expr: {
    $gt: ["$salary", { $multiply: ["$experience", 500000] }]
  }
})
```

**Example 2 — Experience less than salary ÷ 10 lakh:**
```js
db.jobs.find({
  $expr: {
    $lt: ["$experience", { $divide: ["$salary", 1000000] }]
  }
})
```

**$expr Operators:**

| Operator | Meaning |
|----------|---------|
| `$gt` | Greater than |
| `$lt` | Less than |
| `$eq` | Equal |
| `$multiply` | Multiplication |
| `$divide` | Division |
| `$add` | Addition |

> 💡 `$expr` runs **per document** — powerful but slower than normal queries. Use only when fixed-value filters are not enough.

---

## 🔗 Combined Real-World Query

Mid-level Backend jobs in Bangalore with salary above 10 LPA:

```js
db.jobs.find({
  role: { $regex: "Backend", $options: "i" },
  experience: { $gte: 3, $lte: 5 },
  location: "Bangalore",
  salary: { $gt: 1000000 }
})
```

> Uses **regex + range filter + exact match + numeric comparison** — all in one query. This is real backend filtering.

---

## 📌 Quick Reference

| Operator | Use When | Example |
|----------|----------|---------|
| `$not` | Negate a single field condition | Experience NOT > 2 |
| `$nor` | Exclude multiple conditions | Not Amazon AND not Bangalore |
| `$regex` | Pattern match on string fields | Role starts with "Senior" |
| `$expr` | Compare two fields in same document | Salary > Experience × 5L |
