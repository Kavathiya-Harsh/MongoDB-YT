# 🎓 MongoDB CRUD — Student Records Manager

> **Video Series: MongoDB in Practice | Episode 2**
> In this project, we build a real **Student Records Manager** and perform all four CRUD operations step by step.

---

## 📁 Project Setup

```js
use studentDB
db.createCollection("students")
```

| Term | Description |
|------|-------------|
| `studentDB` | Our database |
| `students` | Our collection (like a folder of student files) |

---

## ✅ Task 1 — CREATE: Add Students

**Insert one student:**
```js
db.students.insertOne({
  name: "Raj Patel",
  age: 20,
  course: "Computer Science",
  grade: "A",
  enrolled: true
})
```

**Insert multiple students at once:**
```js
db.students.insertMany([
  { name: "Priya Shah",  age: 21, course: "Data Science",      grade: "B", enrolled: true  },
  { name: "Sam Mehta",   age: 19, course: "Web Development",   grade: "A", enrolled: true  },
  { name: "Anika Joshi", age: 22, course: "Computer Science",  grade: "C", enrolled: false }
])
```

> 💡 MongoDB auto-generates a unique `_id` for every document.

---

## 🔍 Task 2 — READ: Find Students

**Get all students:**
```js
db.students.find({})
```

**Filter by course:**
```js
db.students.find({ course: "Computer Science" })
```

**Multiple conditions (grade A + enrolled):**
```js
db.students.find({ grade: "A", enrolled: true })
```

**Sort by name A→Z:**
```js
db.students.find({}).sort({ name: 1 })   // 1 = ascending, -1 = descending
```

**Projection — show only name and course:**
```js
db.students.find({}, { name: 1, course: 1, _id: 0 })
```

> 💡 Projection keeps API responses fast and lightweight by fetching only what you need.

---

## ✏️ Task 3 — UPDATE: Modify Records

**Update a single student (add email + new grade):**
```js
db.students.updateOne(
  { name: "Raj Patel" },
  { $set: { grade: "A+", email: "raj@university.com" } }
)
```

**Update all enrolled students (end of semester):**
```js
db.students.updateMany(
  { enrolled: true },
  { $set: { termCompleted: true } }
)
```

> 💡 `$set` only updates the specified fields — everything else stays untouched.

---

## 🗑️ Task 4 — DELETE: Remove Records

**Delete a single student:**
```js
db.students.deleteOne({ name: "Anika Joshi" })
```

**Delete all unenrolled students:**
```js
db.students.deleteMany({ enrolled: false })
```

> ⚠️ **Safety Rule:** Always run a `find()` with your filter **before** running `deleteMany()` to verify which documents will be affected. Never use an empty filter `{}` in production.

---

## 📌 Quick Reference

| Operation | Method | Example Use Case |
|-----------|--------|-----------------|
| **Create** | `insertOne()` / `insertMany()` | Register new students |
| **Read** | `find()` + `.sort()` + projection | Display student list |
| **Update** | `updateOne()` / `updateMany()` | Update grades, mark term complete |
| **Delete** | `deleteOne()` / `deleteMany()` | Remove withdrawn students |
