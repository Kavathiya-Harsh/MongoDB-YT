# 🍃 MongoDB CRUD Operations

> Study notes based on the video lecture. Master these 8 methods and you can build the data layer of virtually any application.

---

## 📌 What is CRUD?

| Letter | Operation | Description |
|--------|-----------|-------------|
| **C** | Create | Add new data to the database |
| **R** | Read | Fetch data without changing anything |
| **U** | Update | Modify data that already exists |
| **D** | Delete | Permanently remove documents |

> Each operation has two versions — a **One** version (single document) and a **Many** version (multiple documents).

---

## ➕ C — Create

### `insertOne()` — Insert a single document
```js
db.users.insertOne({ name: "Raj", email: "raj@example.com", age: 25 })
```
- Saves one document to the collection
- MongoDB **auto-generates** a unique `_id`

### `insertMany()` — Insert multiple documents
```js
db.users.insertMany([
  { name: "Priya", age: 22 },
  { name: "Sam",   age: 30 },
  { name: "Anika", age: 27 }
])
```
- Pass an **array** of documents in a single call
- Returns the inserted IDs for every document
- ✅ Always prefer `insertMany` for bulk data — it's significantly faster than calling `insertOne` in a loop

---

## 🔍 R — Read

### `findOne()` — Get the first matching document
```js
db.users.findOne({ name: "Raj" })
```
- Returns **one document** or `null` if nothing matches
- Mostly used with `_id` in real APIs

### `find()` — Get all matching documents
```js
// Get all documents
db.users.find({})

// Filter documents
db.users.find({ age: { $gt: 20 } })
```

### Chaining Methods
```js
// Sort by age (ascending), limit to 5 results
db.users.find({}).sort({ age: 1 }).limit(5)
```

### Projection — Return specific fields only
```js
db.users.find({}, { name: 1, email: 1, _id: 0 })
```
- `1` = include the field
- `0` = exclude the field
- Useful when you don't want to send the entire document to the client

### Common Query Operators
| Operator | Meaning |
|----------|---------|
| `$gt` | Greater than |
| `$lt` | Less than |
| `$eq` | Equal to |
| `$in` | Matches any value in an array |

## ✏️ U — Update

### `updateOne()` — Update a single document
```js
db.users.updateOne(
  { name: "Raj" },
  { $set: { email: "newraj@example.com" } }
)
```

### `updateMany()` — Update all matching documents
```js
db.users.updateMany(
  { role: "user" },
  { $set: { plan: "free", active: true } }
)
```
Sets `plan` and `active` fields on **every** matching user in a single operation.

### Useful Update Operators
```js
// $set — change specific fields
{ $set: { email: "new@example.com" } }

// $inc — increment a number (e.g. track login count)
db.users.updateOne({ name: "Raj" }, { $inc: { loginCount: 1 } })

// $unset — remove a field
{ $unset: { oldField: "" } }
```

> ⚠️ **Important:** Always use `$set`. Without it, MongoDB **replaces the entire document** and you lose every other field. This is the most common beginner mistake!

---

## 🗑️ D — Delete

### `deleteOne()` — Delete the first matching document
```js
db.users.deleteOne({ name: "Raj" })
```
- In production, always delete by `_id` to target the exact right document

### `deleteMany()` — Delete all matching documents
```js
// Delete all inactive accounts
db.users.deleteMany({ active: false })
```

> 🚨 **Critical Warning:** Passing an empty object deletes your **entire collection** instantly — no confirmation, no undo!
> ```js
> db.users.deleteMany({})  // ❌ DELETES EVERYTHING
> ```
> **Always run `find()` first**, verify your filter returns the right documents, then delete.

## 📋 Quick Reference — All 8 Methods

| Operation | Single Document | Multiple Documents |
|-----------|-----------------|--------------------|
| **Create** | `insertOne()` | `insertMany()` |
| **Read** | `findOne()` | `find()` |
| **Update** | `updateOne()` | `updateMany()` |
| **Delete** | `deleteOne()` | `deleteMany()` |
