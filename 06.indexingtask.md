## ✅ Task 1 — Single Field Index + explain()

**Goal:** Create an index on `location` and verify MongoDB uses **IXSCAN** instead of **COLLSCAN**.

**Step 1 — Check without index:**
```js
db.jobs.find({ location: "Bangalore" }).explain("executionStats")
```
> 🔴 You will see `"stage": "COLLSCAN"` — MongoDB scans every document.

**Step 2 — Create the index:**
```js
db.jobs.createIndex({ location: 1 })
```

**Step 3 — Verify again:**
```js
db.jobs.find({ location: "Bangalore" }).explain("executionStats")
```
> 🟢 Now you will see `"stage": "IXSCAN"` — MongoDB uses the index directly.

**💡 Key Takeaway:** One `createIndex()` call changes how the entire query executes. Always verify with `explain()`.

## ✅ Task 2 — Compound Index (Filter + Sort)

**Goal:** Create a compound index on `location` and `salaryLPA` to optimize filtering by location and sorting by salary.

**Create the index:**
```js
db.jobs.createIndex({ location: 1, salaryLPA: -1 })
```

**Run the query:**
```js
db.jobs.find({ location: "Bangalore" }).sort({ salaryLPA: -1 })
```

**Verify with explain:**
```js
db.jobs.find({ location: "Bangalore" }).sort({ salaryLPA: -1 }).explain("executionStats")
```
> 🟢 You will see IXSCAN with no in-memory sort — both filtering and sorting handled by the index.

**⚠️ Prefix Rule:**

| Query | Uses This Index? |
|-------|-----------------|
| Filter by `location` only | ✅ Yes |
| Filter by `location` + sort by `salaryLPA` | ✅ Yes |
| Filter by `salaryLPA` only | ❌ No |

> 💡 Field order in a compound index matters. Always match it to your real query patterns.

## ✅ Task 3 — Multikey Index (Array Field)

**Goal:** Create an index on the `skills` array and find all jobs requiring **"MongoDB"** as a skill.

**Create the index:**
```js
db.jobs.createIndex({ skills: 1 })
```

**Run the query:**
```js
db.jobs.find({ skills: "MongoDB" })
```
> Returns: **Flipkart** and **Swiggy** — both have MongoDB in their skills array.

**Verify with explain:**
```js
db.jobs.find({ skills: "MongoDB" }).explain("executionStats")
```
> 🟢 IXSCAN — MongoDB used index entries per array element, not a manual array scan.

**💡 Key Takeaway:** When you index an array field, MongoDB creates a separate index entry for each element. This is called a **Multikey Index** and it's automatic.

## ✅ Task 4 — Text Index (Full-Text Search)

**Goal:** Create a text index on the `role` field and search for jobs containing the keyword **"Backend"**.

**Create the index:**
```js
db.jobs.createIndex({ role: "text" })
```

**Search using $text:**
```js
db.jobs.find({ $text: { $search: "Backend" } })
```
> Returns: **Backend Developer (Flipkart)** and **Senior Backend Engineer (Amazon)**

**💡 Text Index on Multiple Fields:**
```js
db.jobs.createIndex({ role: "text", company: "text" })
```
> One index covers text search across both fields.

**⚠️ Important Rule:** Only **one text index** is allowed per collection. Include all text-searchable fields in a single index.

## ✅ Task 5 — hint() + explain() Analysis

**Goal:** Force MongoDB to use a specific index using `hint()` and verify the execution plan.

**Force a specific index:**
```js
db.jobs.find({ location: "Bangalore" })
  .hint({ location: 1 })
  .explain("executionStats")
```
> MongoDB will use the `location` index specifically, even if a compound index is available.

**When to use hint():**

| Situation | Use hint()? |
|-----------|------------|
| Routine production queries | ❌ No — let MongoDB decide |
| Debugging slow queries | ✅ Yes |
| Comparing two index strategies | ✅ Yes |
| Performance testing | ✅ Yes |

## 📊 COLLSCAN vs IXSCAN — Quick Reference

| Feature | COLLSCAN | IXSCAN |
|---------|----------|--------|
| Scans | Every document | Only matching index entries |
| Speed | Slow on large data | Fast and efficient |
| Resource Usage | High CPU & memory | Low CPU & memory |
| Production Ready | ❌ No | ✅ Yes |
| How to check | `.explain("executionStats")` | `.explain("executionStats")` |

## 📌 All Index Types — Summary

| Index Type | When to Use | Example |
|------------|-------------|---------|
| **Single Field** | Filter/sort on one field | `{ location: 1 }` |
| **Compound** | Filter + sort on multiple fields | `{ location: 1, salaryLPA: -1 }` |
| **Multikey** | Field contains an array | `{ skills: 1 }` |
| **Text** | Keyword search on string fields | `{ role: "text" }` |
| **Hashed** | Equality lookups in sharded clusters | `{ rollNumber: "hashed" }` |
| **Geospatial** | Location-based distance queries | `{ location: "2dsphere" }` |
